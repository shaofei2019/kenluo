依赖注入的两种方式

发表于 2020-07-17 更新于 2020-07-23 分类于 Spring 阅读次数：
本文字数： 7k 阅读时长 ≈ 6 分钟

一 什么是依赖注入

依赖注入是一个过程，通过该过程可以解决对象依赖问题。现假设一个场景，有类A和类B，类A依赖类B，以往在我们实例化类A时，需要先拥有类B的实例化对象，利用这个对象才能完成类A的实例化。现在，类B的实例化任务由IoC容器接管了，我们可以直接获取类A的实例化对象，而无需顾及类B的实例化过程，我们需要做的仅是类A和类B的依赖关系的声明。

二 依赖注入的两种方式

有两种依赖注入的方式，其一基于非静态的setter方法，其二基于构造方法，下面列举两个实例。

1 setter-based DI

实例化类和该类的相关依赖：

//Person.java
package com.example.demo.pojo;

public class Person {
    private Integer id;
    private Cat cat;
    private Car car;

   /* public Person(Integer id, Cat cat, Car car) {
        this.id = id;
        this.cat = cat;
        this.car = car;
    }*/

    public void setId(Integer id) {
        this.id = id;
    }

    public void setCat(Cat cat) {
        this.cat = cat;
    }

    public void setCar(Car car) {
        this.car = car;
    }

    public void catWork(){
        cat.work();
    }

    public void carWork(){
        car.work();
    }
}

//Cat.java
package com.example.demo.pojo;

public class Cat {
    public void work(){
        System.out.println("喵喵喵。。。");
    }
}
//Car.java
package com.example.demo.pojo;

public class Car {
    public void work(){
        System.out.println("汽车行驶中。。。");
    }
}

配置文件：

<bean id="person" class="com.example.demo.pojo.Person">
        <property name="id" value="1" />
        <property name="car" ref="car" />
        <property name="cat" ref="cat" />
</bean>
<bean id="cat" class="com.example.demo.pojo.Cat"/>
<bean id="car" class="com.example.demo.pojo.Car"/>

测试方法:

public void test1(){
        ApplicationContext context = new ClassPathXmlApplicationContext("spring.xml");
        Person person = (Person) context.getBean("person");
        person.catWork();
        person.carWork();
    }

2 constructor-based DI

实例化类和该类的相关依赖：

//Person.java
package com.example.demo.pojo;

public class Person {
    private Integer id;
    private Cat cat;
    private Car car;

    public Person(Integer id, Cat cat, Car car) {
        this.id = id;
        this.cat = cat;
        this.car = car;
    }

    public void catWork(){
        cat.work();
    }

    public void carWork(){
        car.work();
    }
}
//Cat.java
package com.example.demo.pojo;

public class Cat {
    public void work(){
        System.out.println("喵喵喵。。。");
    }
}
//Car.java
package com.example.demo.pojo;

public class Car {
    public void work(){
        System.out.println("汽车行驶中。。。");
    }
}

配置文件：

<bean id="person" class="com.example.demo.pojo.Person">
        <constructor-arg type="java.lang.Integer" value="1" />
        <constructor-arg ref="cat" />
        <constructor-arg ref="car" />
</bean>
<bean id="cat" class="com.example.demo.pojo.Cat"/>
<bean id="car" class="com.example.demo.pojo.Car"/>

测试方法:

public void test1(){
        ApplicationContext context = new ClassPathXmlApplicationContext("spring.xml");
        Person person = (Person) context.getBean("person");
        person.catWork();
        person.carWork();
    }

三 区别

两者可以混合使用，可将Constructor-based用于强制性依赖项，Setter-based用于可选性依赖性。可在setter方法上标注@Required注解，使其成为必需依赖项。

四 循环依赖问题

假设一个会出现循环依赖问题的场景，类A通过构造方法注入类B的实例，类B通过构造方法注入类A的实例。如果你为类A和类B的相互注入而配置了bean，则Spring IoC容器回在运行时检测到循环引用，抛出BeanCurrentlyInCreationException异常，如下所示。

WARN org.springframework.context.support.ClassPathXmlApplicationContext - Exception encountered during context initialization - cancelling refresh attempt: org.springframework.beans.factory.BeanCreationException: Error creating bean with name 'a' defined in class path resource [spring.xml]: Cannot resolve reference to bean 'b' while setting constructor argument; nested exception is org.springframework.beans.factory.BeanCreationException: Error creating bean with name 'b' defined in class path resource [spring.xml]: Cannot resolve reference to bean 'a' while setting constructor argument; nested exception is org.springframework.beans.factory.BeanCurrentlyInCreationException: Error creating bean with name 'a': Requested bean is currently in creation: Is there an unresolvable circular reference?

解决这个问题的方案是，修改类A和类B的依赖注入方式，从构造方法注入修改为setter注入，实例如下。 相关类：

//A.java
package com.example.demo.pojo;

public class A {
    public String name;
    public B b;

    /*public A(String name, B b) {
        this.name = name;
        this.b = b;
    }*/
    public void setName(String name) {
        this.name = name;
    }
    public void setB(B b){
        this.b = b;
    }

    public void work(){
        System.out.println("FROM-A: A.name = " + this.name + " B.name = " + b.name);
    }
}

//B.java
package com.example.demo.pojo;

public class B {
    public String name;
    public A a;

  /*  public B(String name, A a) {
        this.name = name;
        this.a = a;
    }*/
  public void setName(String name) {
      this.name = name;
  }
    public void setA(A a){
        this.a = a;
    }

    public void work(){
        System.out.println("FROM-B: B.name = " + this.name + " A.name = " + a.name);
    }
}

配置文件：

<bean id="a" class="com.example.demo.pojo.A">
        <property name="name" value="A"/>
        <property name="b" ref="b"/>
    </bean>
    <bean id="b" class="com.example.demo.pojo.B">
        <property name="name" value="B"/>
        <property name="a" ref="a"/>
    </bean>

测试方法：

public void test1(){
        ApplicationContext context = new ClassPathXmlApplicationContext("spring.xml");
        A a = (A)context.getBean("a");
        a.work();
        B b = (B)context.getBean("b");
        b.work();
    }

测试结果：
